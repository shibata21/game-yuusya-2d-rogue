# imagegen素材の受け入れ基準

## 目的

外部生成した素材を取り込むとき、見た目の問題をテスト緩和で隠さず、ゲーム内の48px表示まで含めて仕様どおりか判定する。

## 基本原則

- 既存テストの失敗は、まず生成物または実装の不備の兆候として扱う。テストを通すためだけに閾値を緩めない。
- テストを変更できるのは、利用者が仕様を明示的に変更した場合だけとする。変更時は、以前の検査がなぜ不要になったか、代わりに何を保証するかを同じ変更内に記録する。
- 原画が見た目どおりでも、トリミング・透過化・正規化・48px縮小後に崩れる場合がある。受け入れ判定は生成原画と最終アトラスの両方で行う。
- 素材の再生成が必要なら、失敗した検査を維持し、`pass_to_codex/` に原因、再生成条件、数値基準を書く。失敗した素材をpushしない。

## 画像ごとの確認

### 透過素材

- 単色クロマキーは透過化後に四隅が透明であること、不透明なキー色が残っていないことを検査する。
- 影、発光、粒子、煙などが輪郭の外に出ると、正規化時に本体が縮小する。表示サイズを揃える必要がある素材では、装飾を輪郭内に収める。
- 同一シルエットを要求する差分は、原画セルと最終48pxセルの不透明領域の幅・高さを比較する。

### スプライトと方向

- 攻撃、採掘、詠唱、回避などの効果は、キャラクターの向きと同じ方向へ進むことを原画と最終アトラスで確認する。
- 実行時に左右反転する方向は、反転後の見た目も確認する。原画側の補正と実行時の補正を重ねて反転させない。

### タイル

- セル区切り、隣接セルの混入、発光外周は、並べたときの線や縁として現れる。原画セルの外周をトリムした後の48pxタイルで、全辺の明度差を検査する。
- 進化段階の違いは、仕様で許可されない限り外周効果ではなくタイル内部の模様で表現する。

## 受け入れ手順

1. 依頼文の対象、方向、行列順、背景色、禁止要素をマニフェストまたは再生成依頼に記録する。
2. 生成原画を目視し、セル配置、向き、輪郭外の装飾、キー色残りを確認する。
3. 素材ビルド後、最終48pxアトラスの該当セルと実行時に反転するセルを確認する。
4. `npm run assets:check`、`npm test`、`npm run build`、`npm run smoke` を実行する。
5. 失敗が出た場合は、失敗条件が現行仕様を正しく表すかを先に確認する。仕様が変わっていなければ、素材または実装を直す。
6. 全検証が通り、差分が対象素材と必要な生成物・テストに限定されてからcommit・pushする。

## 2026-07-13の卵シート事例

3x3卵シートは、通常・第一進化・第二進化で同じ中立卵形を使う仕様だった。第一進化行に輪郭外のリングと粒子が生成され、正規化後の卵幅が通常31px、第一進化25px、第二進化33pxになった。

原画セル全体の不透明領域の高さを比較するテストは、この問題を検出していた。進化段階の外周効果を許容する根拠はなかったため、テストの緩和は誤りと判断する。再生成では、進化模様を卵の輪郭内に収め、9セルの外形寸法を揃える。
